How URL matching works:
  • Simple text matching - Enter any part of the URL (e.g., "github.com", "drive.google", "dropbox")
  • No need for https:// - Just the domain or path is enough
  • One filter per entry - Create separate filters for multiple domains
  • First match wins - If a URL matches multiple filters, the first one is used
Examples: github.com matches any GitHub download drive.google.com matches Google Drive files amazonaws.com matches AWS S3 downloads
